# Utility-Site Portfolio — Implementation Plan
*Hermes-Agent-driven build, deploy, and SEO workflow*

This is the operational companion to `utility_site_business_plan.md`. It turns the strategy into a repeatable, mostly-automated pipeline.

---

## 0. Prerequisites (one-time)

```bash
# Install Hermes Agent
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
hermes setup                       # pick LLM provider

# Confirm required toolsets are enabled
hermes tools list                  # ensure web + terminal are on
hermes tools enable web terminal

# Accounts needed
#  - GitHub  (GITHUB_TOKEN in ~/.hermes/.env)
#  - Domain registrar API (Namecheap / Porkbun / Dynadot)
#  - Static host CLI token (Netlify / Vercel / Cloudflare)
#  - Google AdSense + Search Console
```

Create a tracking sheet `keywords.xlsx` with columns:
`keyword | search_volume | KD | gap_flag | domain | repo_url | live_url | status`

---

## 1. Pipeline Overview

```
Keyword research
   → SERP gap check (Hermes)
   → Generate site (Hermes prompt → index.html, style.css, README.md)
   → Create GitHub repo + push
   → Register domain (registrar API)
   → Deploy to static host
   → Insert AdSense snippet
   → SEO optimisation (Hermes skill)
   → Publish remixable template
   → Monitor traffic + revenue → iterate
```

Every step except **domain purchase** and **AdSense copy/paste** is fully automatable via Hermes.

## 2. Phase 1 — Idea Generation & Validation

1. Collect seed keywords into `keywords.xlsx`.
2. Filter for **search volume ≥ 10k/mo** and **KD ≤ 10**.
3. Run the SERP gap check:

```bash
hermes chat -q "Search Google for '<KEYWORD>'. Are there sites on the first page with a low trust score (new/thin sites)? Return only YES if a gap exists, else NO."
```

4. Mark `gap_flag = YES` rows and pick 3–5 per week.

## 3. Phase 2 — Site Generation (Hermes)

### 3.1 Prompt template — `prompts/create_utility.txt`

```
You are a web-developer assistant.
Goal: Build a static, mobile-responsive utility website for the keyword "<KEYWORD>".
Requirements:
1. HTML5 page with a prominent input field that works instantly (no extra button if possible).
2. Optional filters (start-letter, length) as collapsible controls.
3. Minimal, accessible CSS.
4. A 150-word "What this tool does" section containing the primary keyword.
5. FAQ schema (JSON-LD) with three Q&A derived from the keyword.
6. Google AdSense placeholder: <!-- ADSENSE_PLACEHOLDER -->
7. README.md explaining how to replace the placeholder with real ad code.
8. Output three files: index.html, style.css, README.md.
```

### 3.2 Run

```bash
hermes chat -q "$(cat prompts/create_utility.txt | sed 's/<KEYWORD>/word unscrambler/g')" > /tmp/hermes_output.txt
```

Write the returned code blocks to `index.html`, `style.css`, `README.md` (Hermes can do this directly via `write_file`, or copy-paste manually).

## 4. Phase 3 — GitHub Repo + Push

`scripts/create_repo_and_push.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
REPO="$1"            # e.g. word-unscrambler-utility
OWNER="harveyai26032026-ai"

# 1. Create repo via API
curl -s -X POST -H "Authorization: token $GITHUB_TOKEN" \
  -d "{\"name\":\"$REPO\",\"description\":\"Free utility tool — generated by Hermes Agent\",\"private\":false,\"auto_init\":true}" \
  https://api.github.com/user/repos > /dev/null

# 2. Clone, copy, commit, push
TMP="/tmp/$REPO"
rm -rf "$TMP"
git clone "https://$GITHUB_TOKEN@github.com/$OWNER/$REPO.git" "$TMP"
cp index.html style.css README.md "$TMP/"
cd "$TMP"
git add .
git commit -m "Initial site generated by Hermes Agent"
git push origin main
echo "Repo: https://github.com/$OWNER/$REPO"
```

## 5. Phase 4 — Domain Registration (low-cost)

| Provider | Cost | API call (via Hermes terminal) |
|----------|------|-------------------------------|
| Porkbun | ~$9/yr | `curl -s -u $PORKBUN_API: -X POST https://porkbun.com/api/json/v3/domain/create -d '{"domain":"wordunscramble.com"}'` |
| Namecheap | $8–$12/yr | Namecheap `domains.create` XML API |
| Dynadot | $7–$10/yr | Dynadot REST `register` endpoint |

After registration, add a DNS A/CNAME record pointing at the static host (set in Phase 5).

If you prefer manual purchase, record the domain in `keywords.xlsx` and continue.

## 6. Phase 5 — Deploy to Free Static Host

### Option A — Netlify (easiest)
```bash
netlify sites:create --name word-unscrambler --repo https://github.com/$OWNER/word-unscrambler-utility.git
```

### Option B — Vercel
```bash
vercel --prod --confirm
```

### Option C — Cloudflare Pages
```bash
curl -s -X POST "https://api.cloudflare.com/client/v4/accounts/$CF_ACCOUNT_ID/pages/projects" \
  -H "Authorization: Bearer $CF_API_TOKEN" -H "Content-Type: application/json" \
  -d '{"name":"word-unscrambler","production_branch":"main","source":{"type":"github","config":{"owner":"'$OWNER'","repo_name":"word-unscrambler-utility"}}}'
```

Pick one host (Netlify for simplicity; Vercel for edge caching; Cloudflare for DNS-level control).

## 7. Phase 6 — Insert AdSense

1. Apply for AdSense (use a primary domain).
2. Copy the auto-ads snippet from the dashboard.
3. Replace the placeholder:

```bash
sed -i 's|<!-- ADSENSE_PLACEHOLDER -->|<YOUR_ADSENSE_SCRIPT>|' index.html
git add index.html && git commit -m "Add AdSense code" && git push
```

4. Verify the ad renders on the live URL after a few minutes.

## 8. Phase 7 — SEO Optimisation Checklist

Bundle into a reusable Hermes skill `seo_optimize_utility_site` (params: domain, keyword).

| Item | Action |
|------|--------|
| Title | `<title>` front-loads primary keyword, ≤ 60 chars |
| Meta description | ≤ 155 chars, keyword + CTA |
| Header hierarchy | One H1 (keyword), logical H2/H3 |
| Body content | 150–200 words, ~1% keyword density, 2 FAQs |
| FAQ schema | JSON-LD `FAQPage`, 3 Q&A |
| Open Graph / Twitter | og:title, og:description, og:image (1200×630) |
| Canonical | `<link rel="canonical" href="https://domain/">` |
| WebApplication schema | `offers: Free`, `operatingSystem: All` |
| Image alt text | Descriptive alt on all images |
| Page speed | Minify CSS/JS, inline critical CSS |
| Mobile-first | viewport meta + responsive flexbox |
| Internal linking | Footer links to other portfolio tools |
| Sitemap + robots | `sitemap.xml` (root URL) + permissive `robots.txt` |
| Search Console | Verify via DNS TXT, submit sitemap |

Invoke once installed:
```bash
hermes chat -q "Run skill seo_optimize_utility_site with domain https://wordunscramble.com and keyword \"word unscrambler\""
```

## 10. Automation — Weekly Batch

`scripts/run_batch.sh` chains the phases for each keyword. Schedule it with a Hermes cron job:

```bash
hermes cron create \
  --schedule "every Monday 09:00" \
  --prompt "Run /home/harvey/output/repo/scripts/run_batch.sh" \
  --name weekly_utility_batch \
  --deliver origin
```

```bash

```
